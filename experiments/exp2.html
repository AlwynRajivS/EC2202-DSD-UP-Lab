Designing a Burglar Alarm
with combinational logic

A house alarm should sound only when the system is armed and an intruder is caught by the window sensor or the motion detector. Below is the full design, from truth table to logic gates — plus a live circuit you can operate yourself.

01

How the design comes together

This is a classic combinational logic problem — the output depends only on the present inputs, with no memory of past states.

1

Identify the inputs and output

Three binary inputs: SYS (system armed?), WS (window sensor tripped?), MD (motion detected?). One binary output: ALARM (siren on?). Each signal is either 0 (false / inactive) or 1 (true / active).

2

Translate the spec into a condition

"Activated if an unauthorized person is detected by a window sensor or motion detector" → WS OR MD. "This has to happen only when the system is activated" → that OR result is only valid AND SYS = 1.

3

Build the truth table

List all 2³ = 8 combinations of SYS, WS, MD and evaluate ALARM for each — this gives us the ground truth to simplify against.

4

Simplify with a Karnaugh map

Plot the 1-outputs on a K-map and group adjacent 1s in powers of two to eliminate redundant variables and reach the minimal Boolean expression.

5

Draw the logic diagram

Convert the minimal expression directly into gates: one 2-input OR gate for the sensors, feeding one 2-input AND gate together with SYS.

03

Karnaugh Map Simplification

Rows = SYS, columns = WS·MD in Gray-code order, so adjacent cells differ by only one bit. The two colored loops show the pair-groupings used to simplify the expression.

← swipe to see the full map →
Group A (solid) → SYS·WS — pairs cells 11 and 10 in the SYS=1 row.
Group B (dashed) → SYS·MD — pairs cells 01 and 11 in the SYS=1 row.
The cell at WS·MD=11 belongs to both loops — that overlap is allowed and expected.
Row SYS=0 is all zeros, so SYS appears in every group — it can never be simplified away.
04

Boolean Expression Derivation

Reading the two K-map groups and combining them (Sum of Products):

Group A (SYS·WS, MD don't-care):  SYS·WS Group B (SYS·MD, WS don't-care):  SYS·MD Sum of Products:  ALARM = SYS·WS + SYS·MD Factor out common term SYS:  ALARM = SYS·(WS + MD) ALARM = SYS · (WS + MD)
